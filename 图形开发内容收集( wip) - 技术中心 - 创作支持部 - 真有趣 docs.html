阴影相关cached shadow

Cached Shadows - CRYENGINE 3 Manual - Documentation

aivclab/CachedShadowMaps: Cached Shadow Map Solution for Unity (github.com)

通过为缓存 shadow 结果减少每帧 shadow caster 的渲染数量，同时在场景中光源环境发生变化时，重新计算

需要实时光影渲染，切 shadow caster 的 drawcall 导致性能降低时香肠大地图场景渲染在开启阴影时的优化方法在 SRP 管线中定义，实现成本较低

剔除与合批相关基于 GPU 的 Culling（Vulkan / GL ES ）

Compute based Culling - Vulkan Guide (vkguide.dev)

使用一个 culling shader 来构建 rendering list

兼容 cs 的硬件平台

用于实现 frustum，occlusion 等传统用 cpu 实现的 culling 操作

全公司项目，需要先考虑目标设备

针对高端手机用户（GPU性能叫强）的设备，经一步优化 CPU 压力

在 Unity 平台定制，周期未定
用 cs 实现遮挡剔除（gles 3.1）

OpenGL ES SDK for Android: Occlusion Culling with Hierarchical-Z (arm-software.github.io)

使用 gles 3.0 提供的 occlusion query 功能来判断 draw call 的可见性，但不支持 instancing，且需要 cpu 的读回操作

兼容 cs 的硬件平台

比较小的室内场景，潜在会发生很多遮挡关系的环境

可以应用于竞技场星封闭式环境的 FPS 类型游戏引擎层定制，待评估





renderpass 相关render graph 管线定制流（for URP）

The render graph system | Core RP Library | 17.0.1 

通过模块化的方式配置渲染管线，目前应用于 HDRP，会在近期的 URP 中集成进来

根据需要定制渲染管线特性，不需要 coding所有需要定制渲染管线业务的项目简单易实现，需要关注平台特有的优化接口是否也同样可以应用
mobile deferred rendering

Deferred shading on mobile - Graphics, Gaming, and VR blog - Arm Community blogs - Arm Community

成熟的移动端（tiled-based GPU）渲染方案

仅推荐 tiled-based 移动 gpu

希望尽可能应用更复杂的实时光照（不要预计算）

所需特性，在 forward lighting 的特性不好实现或成本太高

复杂光照造成的性能与内存占用过高时

所有希望表现丰富的光照环境，包括区域光，多光源，AO，等氛围营造的项目，类似 《神界2》


URP 有完整方案，可以测试后评估


基于 prepass 的 overdraw 优化

Arm Guide for Unity Developers - Optimization opportunities in Unity

通过预先渲染深度，降低 fragment 阶段的 gpu 压力

片元瓶颈时

CPU 与 vertex 阶段不是瓶颈时

有大量透明对象需要渲染时，特效或者植被

强特效表现的产品，例如乌啦啦的战斗画面，或者香肠大世界的树木与植被草地的渲染环境

URP 以集成该方案，可以测试不同关卡的渲染情景动态使用
feature 优化相关bloom 性能优化

MarcusXie3D/FastBloomForMobiles:

专为移动端友好的 bloom 特性方案，测试渲染效率提升 60%，不建议同时开启抗锯齿

针对需要开启 bloom 效果的移动端平台预测乌啦啦向的，比较强调场景沉浸体验画面表现的项目，以及对抗锯齿没有强需求的项目shader 和 render pass 修改，非常小

其他基于 clipmap 的 terrain system 管理方案

Geometry clipmaps: simple terrain rendering with level of detail (mikejsavage.co.uk)

Geometry clipmaps: Terrain rendering using nested regular grids (hhoppe.com)

OpenGL ES SDK for Android: Terrain Rendering with Geometry Clipmaps (arm-software.github.io)

通过 clipmap 以 4 x 方式管理大地形渲染资源

需要构建自定义 terrain system 的基础方案的选择

希望大世界渲染不要随着世界的扩大而内存也随之扩大的情形

大世界相关的游戏项目预计构建成本，测试与风险较大，但是其 clipmap 的思想可以不仅仅用于 terrain

TMP 渲染的优化 - 堆内存

TMP(TextMeshPro) - 堆内存优化

希望优化 TMP 本身产生的 GC 问题的情形对 junk 和字体数据大小敏感的手机项目

基于 MSDF 的字体渲染优化

Chlumsky/msdfgen: Multi-channel signed distance field generator (github.com)

将原本 8 bit 单通道数据保存为多通道 sdf 数据，通过优化 shading 实现字体边缘弯折处的锐利效果

应用于需要较大的屏幕占用的字体，且不能接受单通道 sdf 计算得到的圆弧边缘的情况放置类游戏，聊天对话框，重 UI 设计的项目基于 TMP 扩展，外部使用 msdf 生成工具离线烘焙数据，新增 msdf shader，会影响 shading 性能与内存
